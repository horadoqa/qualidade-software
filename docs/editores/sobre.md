# Editores

Saber usar editores de texto no Linux (como nano, vim ou até VS Code) é importante porque **quase tudo no Linux envolve editar arquivos de texto** — não existe sempre uma interface gráfica para isso.

---

## 🧠 1. No Linux, tudo é configurado por arquivos

Diferente de sistemas mais “clicáveis”, no Linux:

* Configurações do sistema são arquivos
* Usuários são definidos em arquivos
* Rede, permissões, serviços… tudo em texto

👉 Então, para mudar algo, você precisa editar esses arquivos.

---

## ⚙️ 2. Exemplos reais onde você usa editores

Você vai precisar de um editor para:

* Configurar rede ou Wi-Fi
* Ajustar permissões de usuários
* Editar scripts de automação (shell scripts)
* Configurar servidores (Apache, Nginx, etc.)
* Alterar comportamento do sistema

---

## 💻 3. Muitas vezes você não tem interface gráfica

Em servidores Linux (muito comuns na internet):

* Não existe “desktop”
* Só existe terminal

👉 Então ferramentas como GNU nano ou Vim são essenciais.

---

## 🚀 4. Automação e programação dependem disso

Se você quiser:

* Criar scripts
* Automatizar tarefas
* Programar em servidores

Você vai editar arquivos o tempo todo.

---

## 🔐 5. É parte do controle do sistema

Editar arquivos corretamente significa:

* Configurar o sistema do seu jeito
* Resolver problemas sozinho
* Não depender de interfaces limitadas

---

## 🧠 Resumo simples

👉 No Linux, saber editar arquivos = saber mexer no sistema.

Sem isso, você fica muito limitado.

---

## ⚖️ Nano vs Vim 

* **nano** → simples, fácil, ideal para iniciantes 👍
* **vim** → poderoso, mas mais difícil de aprender ⚡
* **vi** → versão básica e mais antiga
