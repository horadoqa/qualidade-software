# Banco de Dados

Um **banco de dados** é um sistema usado para **armazenar, organizar e recuperar informações** de forma eficiente.

Pense nele como um “arquivo inteligente”, onde os dados não ficam soltos — eles são estruturados para facilitar buscas, atualizações e análises.

---

## 🧠 Como funciona

Um banco de dados guarda informações em estruturas organizadas, como:

* tabelas
* registros (linhas)
* campos (colunas)

Por exemplo, um sistema de usuários pode ter:

* Nome
* Email
* Senha

---

## 📦 Exemplos populares

Alguns bancos de dados muito usados:

* MySQL → relacional, muito comum
* PostgreSQL → mais avançado e robusto
* MongoDB → baseado em documentos (JSON)
* SQLite → leve, usado em apps locais

---

## 🔄 Tipos principais

### 📊 Relacionais (SQL)

* Usam tabelas com relações entre si
* Exemplo: MySQL, PostgreSQL
* Linguagem: SQL

Aqui estão alguns dos principais bancos de dados SQL:

### 🗄️ Mais usados

* **MySQL** – Muito popular em aplicações web.
* **PostgreSQL** – Avançado, open source e altamente extensível.
* **Microsoft SQL Server** – Forte integração com produtos da Microsoft.
* **Oracle Database** – Muito usado em grandes empresas.
* **SQLite** – Leve, funciona sem servidor (ideal para apps locais/mobile).
* **MariaDB** – Derivado do MySQL, com melhorias.
* **IBM Db2** – Usado em ambientes corporativos.
* **Amazon Aurora** – Versão otimizada na nuvem da AWS.

### 📄 Não relacionais (NoSQL)

* Mais flexíveis (JSON, chave-valor, etc.)
* Exemplo: MongoDB
* Ideais para dados grandes ou menos estruturados

Aqui estão alguns dos principais bancos de dados NoSQL:

### 🗄️ Mais usados

### 📄 Documentos (JSON-like)

* **MongoDB** – Um dos mais populares, armazena dados em formato parecido com JSON.
* **CouchDB** – Focado em sincronização e facilidade de uso.

### 🔑 Chave-valor

* **Redis** – Extremamente rápido, muito usado como cache.
* **Amazon DynamoDB** – Serviço da AWS com alta escalabilidade.

### 🧱 Colunar (wide-column)

* **Apache Cassandra** – Muito usado em sistemas distribuídos e de grande escala.
* **HBase** – Integrado ao ecossistema Hadoop.

### 🌐 Grafos

* **Neo4j** – Ideal para dados com muitas relações (redes sociais, recomendações).
* **ArangoDB** – Suporta grafos, documentos e chave-valor.

* Sistemas de login
* Aplicativos (apps, sites)

* Jogos

## 💡 Exemplo simples

Sem banco de dados:

* Você perde tudo ao fechar o programa

Com banco de dados:

* As informações ficam salvas e podem ser acessadas depois

* subir um MySQL com um comando
* conectar sua aplicação a ele

---

## 🧾 Resumo

* Banco de dados = lugar organizado para guardar dados
* Permite salvar, buscar, atualizar e excluir informações
* Essencial para praticamente qualquer sistema moderno

---
Bancos de dados SQL são aqueles que utilizam a linguagem **SQL** (Structured Query Language) para armazenar, consultar e gerenciar dados de forma estruturada, geralmente em tabelas com linhas e colunas.
