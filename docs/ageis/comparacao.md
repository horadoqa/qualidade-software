# Comparação entre Scrum e Kanban

Um **comparativo direto entre Scrum e Kanban usando exatamente o mesmo projeto (To-Do App com FE, BE e Banco de Dados)** 👇

---

# 🌐 Projeto: To-Do App (Web)

---

# 🔁 SCRUM (por sprints)

## 📅 Como o trabalho é organizado

O trabalho é dividido em ciclos fixos (**Sprints de 1 a 2 semanas**).

---

## 📦 Sprint 1 (exemplo)

**Objetivo:** autenticação + base do sistema

### 📋 Sprint Backlog

* Modelar banco de dados
* Criar API de login/cadastro
* Criar tela de login

### 📅 Durante a sprint

* Nada muda até a sprint acabar (em geral)
* Equipe foca só no que foi planejado

### 🎯 Resultado

* Usuário consegue se cadastrar e logar

---

## 📦 Sprint 2

**Objetivo:** tarefas (CRUD)

* Criar endpoints de tarefas
* Criar interface de lista de tarefas
* Conectar FE com BE

---

## 📦 Sprint 3

**Objetivo:** ajustes finais

* Melhorar UI
* Testes
* Correções

---

## 💡 Característica principal do Scrum

👉 Trabalho em **blocos fechados (sprints)** com objetivo definido

---

# 📋 KANBAN (fluxo contínuo)

## 🧠 Como o trabalho é organizado

Não existem sprints. Tudo flui continuamente no quadro:

---

## 🟦 A FAZER

* Modelar banco de dados
* Criar API de login
* Criar tela de login
* Criar CRUD de tarefas
* Melhorar UI
* Testes

---

## 🟨 FAZENDO

* API de login (BE)
* Tela de login (FE)

---

## 🟩 FEITO

* Setup do projeto
* Estrutura inicial

---

## 🔁 Como o trabalho acontece

* Uma tarefa termina → entra outra imediatamente
* FE, BE e DB trabalham em paralelo o tempo todo
* Não precisa esperar “fim da sprint”

---

## 💡 Característica principal do Kanban

👉 Trabalho em **fluxo contínuo com limite de tarefas em andamento**

---

# ⚖️ COMPARAÇÃO DIRETA

| Aspecto      | Scrum                    | Kanban             |
| ------------ | ------------------------ | ------------------ |
| Organização  | Sprints (tempo fixo)     | Fluxo contínuo     |
| Planejamento | Fechado por ciclo        | Dinâmico           |
| Mudanças     | Geralmente entre sprints | A qualquer momento |
| Entrega      | Ao fim da sprint         | Contínua           |
| Controle     | Reuniões e rituais       | Quadro visual      |
| Foco         | Objetivo por sprint      | Fluxo de tarefas   |

---

# 🚀 RESUMO SIMPLES

* **Scrum:** trabalha em “blocos” com metas por sprint
* **Kanban:** trabalha como uma “esteira contínua” de tarefas
