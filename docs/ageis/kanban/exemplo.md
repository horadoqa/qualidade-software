## 🔁 Como o trabalho flui

* Você sempre puxa uma tarefa de **“A Fazer”**
* Move para **“Fazendo”** quando começa
* Só começa uma nova quando liberar espaço (respeitando o limite)
* Quando termina, move para **“Feito”**

---

## ⚙️ Exemplo de fluxo real

1. O dev de backend termina o endpoint de login → move para “Feito”
2. Pega “criar endpoint de tarefas” → move para “Fazendo”
3. Front-end pega “tela de login” quando tiver espaço

---

## 💡 O que isso mostra na prática

* Você não precisa esperar “sprint acabar”
* O sistema vai evoluindo continuamente
* O foco é **terminar tarefas e manter fluxo constante**
* Evita muita coisa “em andamento” ao mesmo tempo

---

## 🚀 Resumo

No Kanban:

* o trabalho não é dividido em ciclos fixos
* as tarefas fluem continuamente pelo quadro
* o objetivo é manter o sistema sempre avançando sem sobrecarga
