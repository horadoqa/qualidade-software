# Kanban

**Kanban** é um método visual de gerenciamento de trabalho que ajuda a organizar tarefas, acompanhar o progresso e melhorar o fluxo de trabalho de forma contínua.

Ele é muito usado em desenvolvimento de software, mas também funciona para tarefas do dia a dia.

---

## 🧠 Ideia principal

A base do Kanban é simples:

> **visualizar o trabalho e limitar o que está sendo feito ao mesmo tempo**

---

## 📋 Como funciona

Você usa um **quadro dividido em colunas**, por exemplo:

* **A fazer (To Do)**
* **Fazendo (Doing)**
* **Feito (Done)**

Cada tarefa vira um “cartão” que vai passando pelas colunas conforme evolui.

---

## 🔁 Exemplo simples

Imagine organizar um projeto:

| A fazer             | Fazendo      | Feito          |
| ------------------- | ------------ | -------------- |
| Criar tela de login | API de login | Layout inicial |
| Modelar banco       |              |                |

Conforme você trabalha:

* pega uma tarefa de “A fazer”
* move para “Fazendo”
* quando termina, move para “Feito”

---

## ⚙️ Conceitos importantes

* **Limite de trabalho em progresso (WIP)**
  Você define quantas tarefas podem ficar em “Fazendo” ao mesmo tempo
  👉 evita sobrecarga e aumenta o foco

* **Fluxo contínuo**
  Diferente do Scrum, não há sprints fixos — o trabalho flui continuamente

* **Melhoria contínua**
  Você observa onde está travando e melhora o processo

---

## 🆚 Kanban vs Scrum (resumido)

* **Kanban** → fluxo contínuo, mais flexível
* **Scrum** → ciclos fixos (sprints), mais estruturado

---

## 💡 Onde pode usar

## 🚀 Em resumo

O Kanban ajuda você a:

* enxergar tudo que precisa fazer
* focar no que está em andamento
* evitar acúmulo de tarefas
* melhorar sua produtividade de forma simples e visual
