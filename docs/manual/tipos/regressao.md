# 🎯 Teste de Regressão Manual

O **teste de regressão manual** consiste em reexecutar testes já realizados anteriormente para garantir que alterações no sistema não afetaram funcionalidades existentes.

Esse tipo de teste é essencial após correções de bugs, novas funcionalidades ou atualizações do sistema.

---

## 🎯 O que ele faz?

* Verifica se funcionalidades antigas continuam funcionando
* Detecta impactos colaterais de mudanças no sistema
* Garante a estabilidade após novas versões
* Revalida cenários já testados

---

## 🧪 Exemplo

👉 Após uma nova versão do sistema, validar se o login ainda funciona corretamente, mesmo após alterações em outras partes da aplicação.

---

## 📌 Resumo

No teste de regressão manual, o tester **reexecuta testes conhecidos** para garantir que mudanças não quebrem o que já estava funcionando.
