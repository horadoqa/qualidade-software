🔹 1. Teste de Carga (Load Testing)
Avalia como a aplicação se comporta com um número esperado de usuários simultâneos.
👉 Objetivo: verificar se o sistema aguenta o uso normal.





🔹 4. Teste de Volume (Volume Testing)
Testa o desempenho com grandes quantidades de dados (ex: banco de dados muito grande).
👉 Objetivo: verificar impacto do volume de dados no sistema.


🔹 7. Teste de Capacidade (Capacity Testing)
Determina o número máximo de usuários que o sistema pode suportar mantendo um nível aceitável de desempenho.
👉 Objetivo: planejar limites operacionais.

🔹 8. Teste de Latência
Mede o tempo de resposta da aplicação.
👉 Objetivo: garantir respostas rápidas ao usuário.

🔹 9. Teste de Concorrência
Analisa como múltiplos usuários interagem simultaneamente com o sistema.
👉 Objetivo: detectar problemas de bloqueio, race conditions, etc.
