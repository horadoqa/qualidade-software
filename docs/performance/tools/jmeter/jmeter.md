# 📊 Apache JMeter — Guia Introdutório

## 📌 O que é o JMeter?

O **Apache JMeter** é uma ferramenta open source mantida pela Apache Software Foundation, utilizada para realizar **testes de performance** e **carga** em aplicações web, APIs, bancos de dados e outros serviços.

Ele permite simular múltiplos usuários simultâneos, analisar tempos de resposta e identificar gargalos antes que eles afetem usuários reais.

---

## 🚀 Principais Funcionalidades

* Testes de carga (Load Testing)
* Testes de estresse (Stress Testing)
* Testes de API (REST, SOAP, GraphQL)
* Suporte a múltiplos protocolos (HTTP, HTTPS, FTP, JDBC, etc.)
* Execução distribuída (testes em larga escala)
* Relatórios detalhados com métricas de performance
* Interface gráfica (GUI) e modo CLI (linha de comando)

---

## 🧱 Arquitetura Básica

Um plano de teste no JMeter é composto por:

* **Test Plan**: Estrutura principal do teste
* **Thread Group**: Define usuários virtuais (threads), ramp-up e loops
* **Samplers**: Requisições (HTTP Request, JDBC Request, etc.)
* **Listeners**: Exibição de resultados (gráficos, tabelas)
* **Timers**: Controle de tempo entre requisições
* **Assertions**: Validações de resposta
* **Config Elements**: Configurações compartilhadas
