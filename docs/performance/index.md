# 🚀 O que o teste de performance avalia?

**Teste de performance** é o tipo de teste que avalia como um sistema se comporta sob diferentes condições de carga, volume de usuários e uso de recursos, medindo se ele continua **rápido, estável e responsivo**.

O foco não é verificar se a funcionalidade funciona (isso é teste funcional), mas sim **como ela funciona em termos de velocidade e escalabilidade**.

Ele responde perguntas como:

* O sistema aguenta muitos usuários ao mesmo tempo?
* As páginas carregam rápido?
* A API responde dentro do tempo esperado?
* O sistema fica lento ou cai sob carga?
* Existe consumo excessivo de CPU, memória ou rede?

---

# 🧪 Tipos de testes de performance

## ⚡ Load Testing (Teste de carga)

Avalia o comportamento do sistema com um número esperado de usuários.

👉 Exemplo: 1.000 usuários acessando o login ao mesmo tempo.

---

## 💥 Stress Testing (Teste de estresse)

Leva o sistema além do limite para ver quando ele falha.

👉 Exemplo: 50.000 usuários simultâneos.

---

## 📈 Spike Testing

Avalia picos repentinos de acesso.

👉 Exemplo: de 100 para 10.000 usuários em segundos.

---

## ⏱️ Performance/Response Testing

Mede o tempo de resposta das requisições.

👉 Exemplo: login deve responder em menos de 2 segundos.

---

## 🔁 Endurance Testing (Soak Test)

Avalia o sistema por longos períodos de uso contínuo.

👉 Exemplo: sistema rodando por 8 horas com carga constante.

---

# 🧰 Ferramentas comuns

Algumas ferramentas usadas para testes de performance:

* JMeter
* K6
* Gatling
* Locust
* BlazeMeter

---

# 📊 Métricas importantes

Durante os testes, são analisados:

* ⏱ Tempo de resposta
* 📉 Taxa de erro
* 👥 Número de usuários simultâneos
* 💻 Uso de CPU e memória
* 🌐 Throughput (requisições por segundo)

---

# 🎯 Objetivo do teste de performance

Garantir que a aplicação:

* Funcione bem sob carga real
* Não quebre com alto volume de usuários
* Tenha boa experiência para o usuário
* Seja escalável em crescimento

---

# 🧠 Resumo

👉 Teste de performance garante que a aplicação não só funcione, mas funcione **bem sob pressão**, mantendo velocidade, estabilidade e confiabilidade.
