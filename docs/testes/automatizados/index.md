# Testes Automatizados

Testes automatizados são essenciais para garantir qualidade, estabilidade e confiabilidade em aplicações modernas. Eles ajudam a reduzir regressões, aumentar a cobertura de testes e acelerar o ciclo de desenvolvimento.

## O que você vai encontrar aqui

### 🎭 Playwright

Framework moderno para automação de testes end-to-end desenvolvido pela Microsoft.

**Características principais:**

* Suporte a múltiplos navegadores (Chromium, Firefox e WebKit)
* Execução paralela de testes por padrão
* Auto-wait inteligente (reduz flakiness)
* Suporte a testes de API além de UI
* Execução em modo headless ou com interface visual
* Excelente suporte a CI/CD

---

### 🧪 Cypress

Ferramenta focada em testes end-to-end para aplicações web modernas.

**Características principais:**

* Execução diretamente no navegador
* Debugging fácil com time-travel (visualização do passo a passo)
* Recarregamento automático durante o desenvolvimento
* Sintaxe simples e intuitiva baseada em JavaScript
* Forte integração com aplicações front-end (React, Vue, Angular)

---

### 🤖 Robot Framework

Framework genérico de automação baseado em palavras-chave (keyword-driven).

**Características principais:**

* Sintaxe simples e legível (fácil para não programadores)
* Altamente extensível com bibliotecas em Python e Java
* Suporte a testes de UI, API, banco de dados e mais
* Boa integração com Selenium e Appium
* Forte uso em testes de aceitação (ATDD)

---

### 🧭 Selenium

Um dos frameworks mais antigos e amplamente usados para automação de navegadores.

**Características principais:**

* Suporte a múltiplas linguagens (Java, Python, C#, JavaScript, etc.)
* Compatível com praticamente todos os navegadores
* Grande ecossistema e comunidade ativa
* Flexível para diferentes tipos de testes web
* Base para diversas outras ferramentas de automação

---

### 🥒 Cucumber

Ferramenta que suporta Behavior Driven Development (BDD).

**Características principais:**

* Escrita de testes em linguagem natural (Gherkin)
* Facilita colaboração entre times técnicos e de negócio
* Estrutura baseada em Given / When / Then
* Integração com várias linguagens (Java, JS, Ruby, etc.)
* Excelente para testes de aceitação e requisitos
