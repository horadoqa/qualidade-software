# Allure

O **Allure (framework de testes de software)** é uma ferramenta usada por desenvolvedores e times de QA (Quality Assurance) para **gerar relatórios visuais de testes automatizados**.

Ele não é uma ferramenta de teste em si, mas sim um **gerador de relatórios** que se conecta a testes já existentes.

---

## O que é o Allure?

O Allure é um **framework de reporting de testes** que transforma resultados “crus” de testes automatizados em **relatórios interativos e fáceis de entender**.

Ele funciona com várias linguagens e ferramentas de teste, como:

* Java (JUnit, TestNG)
* Python (Pytest)
* JavaScript (Mocha, Jest)
* entre outras

---

## Para que serve o Allure?

### 1. Gerar relatórios visuais de testes

Em vez de ver logs confusos, você vê:

* testes que passaram (✔)
* testes que falharam (❌)
* testes ignorados (⚠️)

---

### 2. Facilitar a análise de bugs

Ele mostra detalhes como:

* qual etapa do teste falhou
* mensagens de erro
* screenshots (se configurado)
* logs completos

Isso ajuda a identificar problemas mais rápido.

---

### 3. Organizar os testes

O Allure organiza os resultados em categorias como:

* funcionalidades testadas
* severidade de bugs
* histórico de execução

---

### 4. Melhorar a comunicação entre equipes

Times de desenvolvimento e QA conseguem entender os resultados sem precisar olhar código ou logs técnicos complexos.

---

## Exemplo prático

Imagine que você testou um sistema de login:

* Teste 1: login válido → passou ✔
* Teste 2: senha errada → passou ✔
* Teste 3: login vazio → falhou ❌

O Allure mostra isso em um painel bonito com gráficos, detalhes e histórico.

---

## Em resumo

O Allure serve para:
👉 **transformar testes automatizados em relatórios claros, visuais e fáceis de analisar**
