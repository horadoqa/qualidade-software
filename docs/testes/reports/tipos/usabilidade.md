# 🧭 Relatório de Usabilidade (Usability Report)

## 🧾 1. Informações gerais

* **Sistema:** Aplicação Web (Portal do Usuário)
* **Tipo de teste:** Teste de usabilidade
* **Ambiente:** Homologação
* **Data:** 04/05/2026
* **Responsável:** QA / UX Tester
* **Participantes:** 5 usuários simulados

---

## 🎯 2. Objetivo

Avaliar a **facilidade de uso, clareza da interface e eficiência dos fluxos principais** da aplicação, identificando pontos de atrito na experiência do usuário.

---

## 🧪 3. Cenários avaliados

* Cadastro de usuário
* Login e logout
* Recuperação de senha
* Atualização de perfil
* Navegação no dashboard

---

## 📊 4. Resultados gerais

| Métrica                     | Resultado |
| --------------------------- | --------- |
| Taxa de sucesso nas tarefas | 88%       |
| Taxa de erro do usuário     | 12%       |
| Tempo médio de conclusão    | 2m 45s    |
| Satisfação média (1–5)      | 3,8       |

---

## 🧭 5. Observações de usabilidade

### 🟢 Pontos positivos

* Fluxo de login simples e rápido
* Interface limpa e organizada
* Botões principais bem visíveis
* Boa responsividade em dispositivos móveis

---

### 🟡 Problemas identificados

* Usuários tiveram dificuldade para encontrar “Recuperar senha”
* Mensagens de erro pouco claras em formulários
* Falta de feedback visual ao salvar dados

---

### 🔴 Problemas críticos de usabilidade

* Processo de atualização de perfil não indica claramente sucesso ou falha
* Navegação no dashboard exige muitos cliques para ações simples

---

## 🧠 6. Feedback dos usuários

* “Não ficou claro se meu perfil foi salvo.”
* “Demorei para achar onde redefinir a senha.”
* “A interface é bonita, mas falta feedback após ações.”

---

## 📉 7. Principais dificuldades

* Descoberta de funcionalidades (discoverability)
* Falta de mensagens de confirmação
* Fluxos com mais etapas do que o necessário

---

## 🚀 8. Recomendações

* Melhorar mensagens de feedback (sucesso/erro)
* Destacar opção de recuperação de senha na tela de login
* Reduzir número de cliques no dashboard
* Adicionar indicadores visuais de carregamento e salvamento
* Realizar testes com mais usuários reais (UX research)

---

## 📌 9. Conclusão

A aplicação possui **boa base de interface e navegação**, mas apresenta problemas de **clareza e feedback ao usuário**, impactando a experiência geral.

👉 **Nível de usabilidade geral: BOM, com melhorias necessárias**

---

## 💡 Resumo

Um **Usability Report** serve para:

* avaliar facilidade de uso do sistema
* identificar pontos de confusão do usuário
* melhorar experiência (UX)
* reduzir erros e frustração
* tornar o sistema mais intuitivo
