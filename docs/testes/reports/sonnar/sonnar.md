# 🧠 O que é o SonarQube?

O **SonarQube** é uma ferramenta usada para **análise de qualidade de código-fonte** em projetos de software.

Ele analisa automaticamente o código e encontra problemas como:

* bugs 🐛
* vulnerabilidades de segurança 🔐
* “code smells” (má práticas de programação)
* duplicação de código
* complexidade excessiva

---

# 🎯 Para que serve?

O Sonar serve para ajudar times de desenvolvimento a:

## ✔️ Melhorar a qualidade do código

Ele mostra onde o código está ruim ou pode ser melhorado.

## ✔️ Detectar bugs cedo

Antes mesmo de o sistema ir para produção.

## ✔️ Aumentar segurança

Identifica falhas como:

* SQL Injection
* exposição de dados
* uso inseguro de APIs

## ✔️ Padronizar código

Garante que todo o time siga boas práticas.

---

# ⚙️ Como ele funciona?

1. O desenvolvedor faz o commit do código
2. O Sonar analisa automaticamente (geralmente no CI/CD)
3. Ele gera um **relatório com problemas encontrados**
4. O time corrige antes do deploy

---

# 📊 O que ele mostra no relatório?

* Quantidade de bugs
* Vulnerabilidades
* Code smells
* Cobertura de testes (%)
* Duplicação de código (%)
* Nota geral de qualidade (A até E)

---

# 🚀 Exemplo prático

Um projeto pode ter resultado assim:

* Bugs: 3
* Vulnerabilidades: 1
* Code smells: 25
* Cobertura de testes: 72%
* Nota geral: **B**

---

# 💡 Resumo

O **SonarQube** é uma ferramenta que:
👉 analisa código automaticamente
👉 encontra problemas de qualidade e segurança
👉 ajuda a manter o software mais confiável
