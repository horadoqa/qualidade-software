# 🧪 O que é Teste Manual?

**Teste manual** é a forma de testar um software em que o **tester executa os cenários diretamente**, sem o uso de ferramentas de automação, simulando o comportamento do usuário final.

Ou seja, tudo é feito “na mão”: clicar, preencher campos, navegar entre telas e validar os resultados visualmente e funcionalmente.

É o processo de verificar se uma aplicação funciona corretamente **executando os testes passo a passo de forma humana**, sem scripts automatizados.

👉 O objetivo é encontrar bugs observando o sistema em uso real.

---

# 🎯 Características do teste manual

* ✔ Executado por um humano (QA/Tester)
* ✔ Não utiliza automação
* ✔ Baseado em cenários de teste
* ✔ Validação visual e funcional
* ✔ Flexível para mudanças rápidas

---

# 🧭 Quando usar teste manual?

O teste manual é ideal quando:

* A funcionalidade está em desenvolvimento
* O sistema muda com frequência
* É necessário testar UX/UI
* Testes exploratórios são necessários
* Não há tempo para automatizar

---

# 🔍 Tipos de teste manual

- 🧭 Teste exploratório
- 📋 Teste funcional manual
- 🎯 Teste de regressão manual
- 🧑‍💻 Teste de usabilidade

---

# ⚖️ Vantagens

* Rápido para começar
* Não exige programação
* Flexível para mudanças
* Bom para encontrar bugs inesperados

---

# ⚠️ Desvantagens

* Mais lento em larga escala
* Sujeito a erro humano
* Difícil de repetir exatamente
* Pouco escalável

---

# 🆚 Teste manual vs automatizado

| Manual                | Automatizado         |
| --------------------- | -------------------- |
| Humano executa        | Código executa       |
| Flexível              | Repetível            |
| Lento                 | Rápido               |
| Ideal para exploração | Ideal para regressão |

---

# 🧠 Resumo

👉 Teste manual é a execução direta dos testes por uma pessoa, simulando o uso real do sistema para identificar falhas, validar regras e garantir qualidade.
