# Postman

Postman é uma ferramenta muito usada por desenvolvedores para **testar, documentar e interagir com APIs** (interfaces que permitem comunicação entre sistemas).

Em termos simples: o Postman permite que você “converse” com um backend sem precisar criar uma interface (como um site ou app) para isso.

---

### 🔧 Para que ele serve?

Com o Postman você pode:

* Enviar requisições HTTP (GET, POST, PUT, DELETE etc.)
* Testar endpoints de APIs
* Ver respostas do servidor (JSON, XML, etc.)
* Simular dados enviados por um app ou site
* Automatizar testes de API

---

### 📌 Exemplo prático

Imagine uma API de usuários:

* `GET /users` → lista usuários
* `POST /users` → cria um usuário

No Postman, você pode:

1. Escolher o método (ex: POST)
2. Informar a URL
3. Enviar um corpo (JSON com dados do usuário)
4. Ver a resposta do servidor

---

### 🧠 Por que ele é tão usado?

* Não precisa escrever código para testar APIs
* Interface visual fácil de usar
* Ajuda no desenvolvimento e depuração
* Permite salvar coleções de requisições
* Possui recursos de automação de testes

---

### 🚀 Resumindo

O Postman é como um “laboratório” para APIs: você testa tudo antes de integrar com seu sistema de verdade.
